Gemini 3.5 Flash — 개요
New

핵심 포지셔닝

"Sonnet급 성능을 Haiku급 가격과 속도로" — 이전 세대 플래그십(3.1 Pro)을 에이전트/코딩에서 초과하면서 25% 저렴.

  • 출력 속도 ~289 tok/s (프론티어 모델 중 최고 수준)
  • 1M 토큰 컨텍스트 윈도우
  • 멀티모달 네이티브 입력 (텍스트, 이미지, 비디오, 오디오, PDF)
  • Gemini 앱 & Google 검색 AI 모드 기본 모델로 즉시 적용
Note

주의사항

마이그레이션 시 반드시 확인해야 할 함정들.

  • thinking_level 함정: 기본값이 high → medium으로 하락 — 명시 설정 필수
  • 토큰 과다 사용: 평균 대비 2배 많은 토큰 소비 → 실질 비용 상승
  • 가격 인상: Gemini 3 Flash 대비 3배, Flash-Lite 대비 6배
  • 미지원: Computer Use, 이미지/오디오 생성, Live API

강점

Gemini 3.5 Flash가 이기는 곳

  • 에이전트/도구 사용: MCP Atlas 전 모델 중 1위 (Opus, GPT-5.5 포함)
  • 멀티모달: MMMU-Pro 84.2% 역대 최고, 비디오/오디오 네이티브 처리
  • 속도: Sonnet 대비 ~5배, Opus 대비 ~10배
  • 가격 대비 성능: Sonnet 절반 가격, GPT-5.5의 1/5 가격
  • 컨텍스트: 1M 토큰 (Claude 200K의 5배)
약점

Gemini 3.5 Flash가 지는 곳

  • 추상 추론: ARC-AGI-2 72.1% — GPT-5.5(84.6%)에 12pt 뒤짐
  • 프로덕션 코딩: SWE-Bench에서 Claude Sonnet 4.6이 선두
  • 장문 검색: 128K에서 GPT-5.5(94.8%)에 17pt 차이
  • 토큰 효율: 평균 2배 많은 토큰 사용 → 실질 비용 상승
Gemini Omni — 세계 모델(World Model) 기반 비디오 생성

Gemini 3.5 Flash가 텍스트를 출력하는 추론 모델이라면, Gemini Omni는 비디오를 출력하는 생성 모델입니다. 완전히 다른 모델 계열입니다.

Omni의 작동 원리: 3개 엔진의 통합

기존 비디오 모델(Sora, Runway 등)은 텍스트 해석 → 이미지 생성 → 프레임 보간이 각각 별개 모델이라 의도 손실·물리 무지·일관성 부족 문제가 있었습니다. Omni는 Google DeepMind의 3개 시스템을 하나의 학습 그래프 안에서 통합했습니다.

Engine

Gemini — 추론 엔진

언어, 의도, 물리, 문화, 역사, 과학을 이해하는 두뇌.

  • 캐릭터 이미지, 배경 프롬프트, 오디오 트랙 사이의 의미론적 관계를 생성 전에 먼저 추론
  • "추론 후 생성(Reason-then-Generate)" 아키텍처
Engine

Veo — 렌더링 엔진

프레임 단위 생성 품질, 모션, 해상도를 렌더링하는 손.

  • Omni Flash: 1080p 5초 프리뷰를 15초 이내 생성
  • 시각 + 음향을 한 번의 디퓨전 패스에서 동시 생성
Engine

Genie — 세계 시뮬레이션

객체, 환경, 물리가 시간에 따라 어떻게 변하는지 시뮬레이션하는 물리 직관.

  • 중력, 운동 에너지, 유체 역학의 직관적 이해
  • 유리잔이 떨어지면 궤적과 충격을 예측 (이미지 추측이 아님)
  • 단, 실제 물리 엔진이 아닌 "물리적 직관을 가진 예측"
대화형 편집 & 사양

영속적 세계 상태 메모리(Persistent World-State Memory): 모델이 장면의 3D 상태를 내부적으로 유지하여, "배경을 바꿔줘" → "카메라를 오른쪽으로" → "비를 추가해줘"처럼 대화하듯 점진적 수정이 가능합니다. 매번 처음부터 새로 생성하는 것이 아닙니다.

Spec

생성 사양

  • 클립 길이: ~10초 (확장 예정)
  • 해상도: Flash 변형 1080p
  • 변형: Omni Flash (빠른) / Omni Pro (고품질)
  • 입력: 텍스트, 이미지, 비디오, 음성 레퍼런스
  • 출력: 비디오 (이미지·오디오는 향후)
  • 워터마크: SynthID 자동 포함
Edit

대화형 편집

  • 모든 지시가 이전 지시 위에 누적
  • 배경 변경, 카메라 앵글 전환, 장면 물리 수정
  • 객체 제거/교체, 업로드 비디오 리믹스
  • 캐릭터·환경·소품의 시각적 일관성 자동 유지
한계

제한사항

  • 크레딧 소비: 2회 생성으로 AI Pro 일일 할당량의 ~86% 소모
  • 스타일 편향: 포토리얼리스틱에 치우침
  • 캐릭터 일관성: 다중 컷에서 불안정할 수 있음
  • 감정 표현: "기술적으로 놀랍지만 감정적으로 평평"
  • 음성 편집: 아직 테스트 중 (책임있는 활용 검토)
  • 내부 아키텍처: Google 미공개, 상세 사양은 추정

Information Agents — 검색이 스스로 찾아오다

기존 검색은 "사용자가 찾으러 가는" 모델이었다면, Information Agent는 24/7 백그라운드 모니터링 → 변화 감지 → 푸시 알림으로 패러다임을 뒤집었습니다.

1
자연어 프롬프트 입력 — AI Mode에서 "만달로리안 근처 극장 티켓 추적해줘" 같은 요청
2
자율 모니터링 — 블로그, 뉴스, 소셜 + 실시간 금융·쇼핑·스포츠 데이터 크롤링
3
합성 알림 — 링크 리스트가 아닌, 여러 소스를 종합한 분석 + 액션 가능한 인사이트 전달
4
히스토리 관리 — 추적 중인 모든 토픽이 AI Mode 히스토리에 보존

활용 예시

  • 항공권 가격 변동 추적
  • 부동산 · 채용 시장 모니터링
  • 스포츠 · 엔터 라이브 이벤트 팔로우
  • 브레이킹 뉴스 실시간 알림
  • 날씨 · 교통 상황 체크

기존 검색과의 차이

  • Pull → Push — 검색하지 않아도 알림 도착
  • 링크 → 합성 분석 — 다수 소스 종합 요약
  • 일회성 → 지속 모니터링 — 24/7 자율 실행
  • 여름 출시 — 미국 Pro/Ultra 구독자 대상

서브에이전트: Gemini CLI vs Antigravity CLI

Gemini CLI에도 서브에이전트가 있었지만, Antigravity CLI에서는 생성 방식, 실행 방식, 자율성이 근본적으로 달라졌습니다.

항목Gemini CLIAntigravity CLI
생성 방식 .gemini/agents/에 마크다운으로 사전 정의 오케스트레이터가 목표 분석 후 동적 생성
호출 사용자가 명시적으로 호출 오케스트레이터가 자율 판단 & 디스패치
실행 동기/순차 — 완료 대기 후 다음 실행 비동기/병렬 — 동시 실행, 터미널 미잠금
컨텍스트 메인 에이전트와 일부 공유/참조 완전 격리 (클린 슬레이트, 부모 히스토리 미상속)

  • 1단계 환경 프로파일링 — 워크스페이스, 권한, 의존성 사전 점검

Managed Agents API 상세

Gemini API에 포함된 서버 사이드 서비스. API 한 번 호출로 Google 서버에 격리 Linux 샌드박스가 프로비저닝되고, 에이전트가 자율적으로 추론·코드 실행·파일 관리·웹 검색을 수행합니다. 개발자가 에이전트 인프라를 직접 구축하지 않아도 됩니다.

API

2개의 API 구조

에이전트 정의(1회)와 실행(매번)이 분리된 구조.

  • Agents API (client.agents.create()) — 에이전트 정의/설정
  • Interactions API (client.interactions.create()) — 에이전트 실행
  • 세션 상태, 대화 이력, 파일 시스템 모두 서버 사이드 관리
Sandbox

샌드박스 환경 사양

OS 레벨 격리. 기본적으로 외부 네트워크·DB·프로덕션 인프라 접근 불가.

  • OS: Ubuntu / CPU: 4코어 / RAM: 16GB
  • 런타임: Python 3.12, Node.js 22
  • GPU/TPU: 미제공 (에이전트 작업용, ML 학습용 아닌)
  • 지속성: 15분 유휴 → 삭제, 7일 미활동 → 완전 삭제
  • 소스 제한: 인라인 1MB/파일, Git 500MB, GCS 2GB
  • 프리뷰 가격: 컴퓨팅 무료, 토큰만 과금
Code

사용 예시

# 한 번의 호출로 에이전트가 샌드박스에서 실행 from google import genai client = genai.Client() interaction = client.interactions.create( agent="antigravity-preview-05-2026", input="Research top 10 AI stories today and create PDF", environment="remote", ) # 멀티턴 — 같은 환경에서 이어서 작업 interaction_2 = client.interactions.create( agent="antigravity-preview-05-2026", environment=interaction.environment_id, # 환경 재사용 previous_interaction_id=interaction.id, # 대화 이력 유지 input="Now create a landing page using JavaScript", )

왜 분리했나?

  • 학습: 컴퓨트 처리량 극대화 필요 (큰 배치, 긴 시퀀스)
  • 추론: 레이턴시 · 메모리 대역폭 극대화 필요 (에이전트 간 실시간 통신)
  • 범용 칩으로는 양쪽 다 차선책 — 전문화가 답
  • "에이전트 시대"에는 추론 비중이 기하급수적으로 증가

TPU 8i 혁신: CAE 엔진

  • Collectives Acceleration Engine — 전용 하드웨어
  • 코어 간 집합연산(All-Reduce) 지연시간 5배 감소
  • MoE · Chain-of-Thought 디코딩에 특화
  • 칩당 TensorCore 2개 + CAE 1개 구성
  • Boardfly 토폴로지 — 네트워크 직경 56% 감소

  • 긴 컨텍스트와 빠른 토큰 출력으로 멀티스텝 작업에 적합
  • 개별 제품의 실제 모델 라우팅은 표면별로 다를 수 있음
Harness

Antigravity 하네스

모델을 "답변기"가 아니라 작업 수행자로 바꾸는 오케스트레이션 층입니다.

  • 목표 분해, 서브에이전트 구성, 병렬 실행
  • CLI·데스크톱·SDK에서 같은 개발자 경험으로 노출
  • 워크플로우, 스킬, 도구 호출을 묶는 실행 프레임
Runtime

Managed Agents / Interactions API

개발자가 직접 VM·브라우저·파일 시스템을 붙이지 않아도 되는 서버 사이드 실행 환경입니다.

  • Agents API로 에이전트 정의, Interactions API로 실행
  • 격리 Linux 샌드박스에서 코드 실행·파일 처리·웹 작업 수행
  • 멀티턴 세션 상태를 런타임이 관리
Dev

AI Studio / Antigravity

에이전틱 앱을 만드는 입구와 본격 개발 표면입니다.

  • AI Studio: 프롬프트 기반 프로토타입, Android 앱 빌드, Workspace 연동
  • Export to Antigravity: 빠른 실험을 개발 워크스페이스로 이전
  • Antigravity: 데스크톱·CLI·SDK로 자동화와 배포 흐름 확장
Consumer

Spark / Search / Workspace / Cart

최종 사용자가 에이전트를 "제품 기능"으로 만나는 표면입니다.

  • Gemini Spark: 개인 작업을 위임하는 24/7 에이전트
  • Search: Information Agents와 AI Mode의 지속 모니터링
  • Workspace: 문서·메일·시트 맥락에서 작업 실행
  • Universal Cart: 쇼핑 탐색, 가격 추적, 구매 흐름 연결
Reading

왜 하나의 스택으로 보이나

발표 표면은 흩어져 있지만, 반복되는 패턴은 동일합니다.

  • 모델이 추론하고, 하네스가 작업을 나누고, 런타임이 실행
  • 개발자는 같은 패턴을 API·CLI·Studio로 제품화
  • 소비자는 Search, Workspace, 쇼핑, Spark에서 결과만 경험
제품 표면별 매핑
표면 사용자에게 보이는 형태 스택에서 읽히는 역할 주의할 점
Gemini 3.5 Flash Gemini 앱·API·에이전트 워크플로우의 모델 선택지 모델 계층 제품별 라우팅과 파라미터는 별도 확인 필요
Antigravity 데스크톱 앱, CLI, SDK 에이전트 하네스 + 개발 표면 로컬 개발 도구와 서버 사이드 API는 역할이 다름
Managed Agents Gemini API 안의 Agents / Interactions 호출 샌드박스 런타임 프로덕션 데이터 접근은 정책·커넥터 설계가 필요
AI Studio 브라우저 기반 앱 제작, Android 빌드, Export 개발자 입구 프로토타입과 운영 배포의 경계가 여전히 중요
Spark / Search / Workspace / Cart 개인 비서, 모니터링, 문서 작업, 쇼핑 흐름 소비자 표면 공식 발표가 아닌 내부 구현 공통성은 분석으로만 보아야 함
전략적 해석

  • 개발 경로 — AI Studio에서 시작해 Antigravity와 API로 확장
  • 배포 표면 — Search, Workspace, Spark, Shopping에 같은 에이전틱 패턴을 적용

읽을 때의 경계선
