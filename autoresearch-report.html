⚙️ 핵심 설계 원칙 ▶

AutoResearch는 3가지 설계 원칙을 기반으로 합니다.

  • 단일 편집 파일: train.py (~630줄) — 에이전트가 이 파일 하나만 수정
  • 단일 스칼라 메트릭: val_bpb (validation bits-per-byte) — 하나의 숫자로 성공/실패 판단
  • 고정 시간 예산: 실험당 5분 — 탐욕적 개선 루프를 안정적으로 유지

🧩 pi-autoresearch 2026 · 오픈소스 ▶

Karpathy AutoResearch 패턴을 도메인에 무관하게 일반화한 pi AI 코딩 에이전트 확장. CLI로 측정 가능한 모든 스칼라 메트릭(테스트 속도, 번들 크기, 빌드 시간, Lighthouse 점수, LLM 훈련 지표 등)에 대해 자율 최적화 루프를 실행합니다.

  • 작성자: David Cortés (@davebcn87) · GitHub ★3,500+ · v1.4.0 (2026.05)
  • 핵심 혁신: 도메인 특화(ML) → 도메인 무관(범용)으로 패턴 일반화
  • 아키텍처: 인프라 계층(도구/영속성/UI) ↔ 도메인 지식(스킬) 분리 설계
  • 도구 체인: init_experiment → run_experiment → log_experiment + 라이프사이클 훅 (before.sh / after.sh)
  • 영속성: autoresearch.jsonl (추가 전용 로그) + autoresearch.md (세션 내러티브 — 새 에이전트가 이전 세션 이어받기 가능)
  • UI: 인라인 위젯 + /autoresearch export 웹 대시보드 + 풀스크린 오버레이 (Ctrl+Shift+F)

평가 메트릭 — 실패 모드별 매핑
4가지 실패 모드를 정확히 탐지하는 메트릭 체계
사용자의 4가지 실패 모드: ❶ 내용이 풍부하지 않음  |  ❷ 빈 근거일 때 내용을 채우지 않음  |  ❸ 근거를 제대로 활용하지 못함  |  ❹ 내용을 반복해서 부풀림
내용이 얼마나 정보로 가득 차 있는지를 측정합니다. 빈 말, 형식적 문장 없이 실질적 주장의 밀도를 평가합니다.
Claim Density
단위 길이당 고유 원자적 주장의 수. 보고서가 실질적 내용을 충분히 담고 있는지 측정.
Claim_Density = |C_unique| / W
|C_unique| = 의미적 중복 제거 후 고유 원자적 주장(atomic claim)의 수. LLM으로 텍스트를 독립 검증 가능한 단위 문장으로 분해한 뒤, 임베딩 코사인 유사도 > 0.85인 것을 병합하여 산출.
W = 보고서의 총 단어 수.
해석: 값이 높을수록 단위 길이당 실질적 정보가 많음. 형식적/반복적 문장이 많으면 하락.

권장 임계값: > 0.05 (단어 100개당 5개 이상 고유 주장)

FActScore
원자적 사실 중 실제로 검증된 비율. 허위/미검증 주장에 페널티 부여.
FActScore(M) = |A_supported| / |A_total|
M = 평가 대상 언어 모델(또는 생성 시스템).
A_total = 생성 텍스트에서 추출한 전체 원자적 사실(atomic fact) 집합.
A_supported = 외부 지식 소스(Wikipedia, KB 등)에 의해 "Supported"로 판정된 사실의 부분집합.
γ = 10 (기본값). 길이 보정 파라미터 — |A_total| < γ이면 페널티를 부과하여 지나치게 짧은 응답의 점수를 감점.

논문: arXiv:2305.14251

Topic Coverage
쿼리에서 기대되는 토픽 대비 실제로 다룬 토픽의 비율.
Topic_Coverage = |T_covered| / |T_expected|
T_expected = 쿼리에서 LLM이 추출한 "완전한 답변에 필요한 하위 토픽" 집합.
T_covered = 생성된 보고서에서 실제로 다룬 토픽 집합 (임베딩 매칭 기반).
해석: 1.0 = 모든 기대 토픽을 다룸. 0.3 = 30%만 커버 → 보고서가 불완전.
Completion Rate (Dr. Bench)
복잡한 다단계 과제에서 서브태스크 완료율.
Completion_Rate = |S_done| / |S_total|
S_total = 쿼리를 분해한 전체 서브태스크 집합.
S_done = 보고서가 실제로 완료(답변)한 서브태스크의 부분집합.

논문: arXiv:2510.02190

검색/참조 근거가 없을 때 공백을 인정하지 않고 임의로 채우거나, 아무것도 쓰지 않는 문제를 탐지합니다.
Context Recall (RAGAS)
참조 답변에 필요한 주장 중 검색된 컨텍스트에 의해 지지되는 비율.
Context_Recall = |C_supported| / |C_ref|
C_ref = 참조 답변(ground truth)에 포함된 전체 주장 집합.
C_supported = 검색된 컨텍스트가 NLI entailment로 지지하는 참조 주장의 부분집합.
해석: 낮은 값 = 검색 시스템이 답변에 필요한 정보를 충분히 가져오지 못함.
Gap Score
보고서가 다루어야 하는 필수 주장 중 실제로 다루지 않은 비율.
Gap_Score = |R_missing| / |R_total|
R_total = 쿼리에서 도출된 필수 정보 항목(required aspects) 집합.
R_missing = 보고서에서 다루지 않은 필수 항목의 부분집합.
해석: 0에 가까울수록 공백 없음. 0.5 이상이면 절반 이상의 필수 주제가 누락.
Self-RAG Critique Tokens
생성 중 자기비판 토큰으로 각 주장의 근거 상태를 실시간 표시.
  • [ISREL] — 검색 결과가 관련 있는가?
  • [ISSUP] — 검색 결과가 주장을 지지하는가?
  • [ISUSE] — 이 정보가 유용한가?
Evidence Sufficiency
쿼리를 하위 질문으로 분해했을 때 실제로 답할 수 있는 비율.
Evidence_Sufficiency = |Q_answerable| / |Q_total|
Q_total = 원본 쿼리를 분해한 하위 질문(sub-question) 집합.
Q_answerable = 검색된 근거만으로 답변 가능한 하위 질문의 부분집합.
용도: 참조 답변 없이도 검색 결과의 충분성을 측정 가능 (reference-free).
검색된 소스가 있음에도 보고서가 이를 무시하거나 잘못 인용하는 문제를 탐지합니다.
Faithfulness (RAGAS)
보고서의 주장 중 실제로 제공된 컨텍스트로 검증 가능한 비율.
Faithfulness = |C_verified| / |C_total|
C_total = 생성된 보고서에서 추출한 전체 원자적 주장 집합.
C_verified = 제공된 컨텍스트(검색 결과)에 의해 NLI entailment로 검증된 주장의 부분집합.
범위: [0, 1]. 1.0 = 모든 주장이 근거에 기반. 0.5 = 절반이 근거 없는 환각.
추천 모델: Vectara HHEM-2.1-Open (T5 기반, LLM 환각 특화) 또는 DeBERTa-MNLI.
NLI 스코어링
자연어 추론 모델로 소스와 생성 텍스트의 함의 관계 측정.
ENT = P(entailment | source, claim)
CON = P(contradiction | source, claim)
DIFF = ENT - CON
ENT = 소스가 주장을 함의(지지)할 확률. [0, 1]. 높을수록 충실.
CON = 소스가 주장을 반박(모순)할 확률. [0, 1]. 높을수록 환각.
DIFF = ENT에서 CON을 뺀 신뢰 점수. [-1, 1]. 양수면 지지, 음수면 환각.
source = NLI 모델의 전제(premise) — 검색된 컨텍스트 문서.
claim = NLI 모델의 가설(hypothesis) — 보고서의 개별 주장.
Citation Precision & Recall
인용의 정확성과 누락 측정.
Citation_Precision = TP / (TP + FP)
Citation_Recall = TP / (TP + FN)
Citation_F1 = 2 × P × R / (P + R)
TP (True Positive) = 올바르게 달린 인용 — 인용된 소스가 실제로 해당 주장을 지지.
FP (False Positive) = 잘못된 인용 — 달았지만 소스가 해당 주장을 지지하지 않음.
FN (False Negative) = 누락된 인용 — 인용이 필요하지만 달지 않음.
Precision: "달린 인용 중 정확한 비율" | Recall: "필요한 인용 중 실제로 단 비율"

ALCE (arXiv:2305.14627) 기반. 임계값: > 0.85

Retrieval Utilization Index (RUI)
검색된 전체 결과 중 실제로 보고서에 반영된 비율.
RUI = |D_used| / |D_retrieved|
D_retrieved = 검색 시스템이 가져온 전체 문서/청크 집합.
D_used = 최종 보고서에 의미적으로 반영된(cos_sim > 0.3~0.4) 문서의 부분집합.
해석: 0.3 = 검색 10건 중 3건만 활용 → 나머지 7건은 검색 비용만 발생, 정보 낭비.
같은 내용을 다른 말로 반복하거나, 섹션 간 중복을 추가해 분량을 부풀리는 문제를 탐지합니다.
Self-Repetition Score (SRS)
문서 내 반복적 표현의 로그 스케일 누적 점수.
SRS(d) = log( Σᵢ₌₁ᵏ (Nᵢ + 1) )
d = 평가 대상 문서.
k = 문서 내 총 4-gram 수.
Nᵢ = i번째 4-gram이 동일 문서 내 다른 위치에 등장하는 횟수 (자기 자신 제외).
+1 = 로그 안정화 (Nᵢ=0일 때 log(1)=0이 되도록).
해석: 값이 높을수록 동일한 표현이 문서 내에서 많이 반복됨.
Self-BLEU
문서 내 단락들 간의 n-gram 중복도. 자기 자신과의 BLEU 점수.
Self-BLEU = (1/n) × Σᵢ BLEU(pᵢ, {pⱼ | j ≠ i})
n = 문서 내 총 단락 수.
pᵢ = i번째 단락 (후보 텍스트로 취급).
{pⱼ | j ≠ i} = i번째를 제외한 나머지 단락들 (참조 텍스트로 취급).
BLEU = n-gram precision의 기하평균 × 길이 보정.
해석: 높은 Self-BLEU = 단락들이 서로 비슷한 표현 사용 → 패러프레이징 반복 의심.
Semantic Novelty per Paragraph
각 단락이 이전 단락들과 의미적으로 얼마나 새로운지 측정.
Novelty(pₖ) = 1 - max{ cos_sim(embed(pₖ), embed(pⱼ)) | j < k }
pₖ = 현재 평가 중인 k번째 단락.
pⱼ (j < k) = k번째 이전에 등장한 모든 단락들.
embed(·) = 문장 임베딩 함수 (예: all-MiniLM-L6-v2, 768차원).
cos_sim = 코사인 유사도. [0, 1] 범위.
해석: 0에 가까움 = 이전 단락과 의미적으로 거의 동일 (반복). 1에 가까움 = 완전히 새로운 정보.
임계값: < 0.15면 심한 반복 의심. 전체 평균 < 0.25면 문서 전체가 반복적.
Information Gain per Section & Redundancy
섹션별 고유 사실 비율 및 전체 중복도.
Info_Gain(sₖ) = |F_new(sₖ)| / |F_total(sₖ)|
Redundancy = 1 - (1/n) × Σₖ Info_Gain(sₖ)
sₖ = k번째 섹션.
F_total(sₖ) = 섹션 sₖ에 포함된 전체 원자적 사실 집합.
F_new(sₖ) = 이전 섹션(s₁..sₖ₋₁)에 없는 신규 사실의 부분집합.
n = 총 섹션 수.
해석: Info_Gain이 0.05 미만인 섹션은 새 정보가 5% 미만 → 통합/제거 대상. Redundancy > 0.4면 심각한 부풀리기.
Info_Gain < 5% → 해당 섹션 통합 또는 제거 대상

Entity Coverage & Relation Coverage
Entity_Coverage = |E_covered| / |E_domain|
Relation_Coverage = |R_extracted| / |R_expected|
E_domain = 해당 도메인의 KB에 존재하는 전체 엔티티 집합.
E_covered = 보고서가 언급하거나 활용한 엔티티의 부분집합.
R_expected = 쿼리 주제에 관련된 기대 관계(relation) 집합.
R_extracted = 보고서에서 실제로 추출/활용된 관계의 부분집합.
KB Quality (종합)
KB_Quality = α × accuracy + β × coverage + γ × consistency
α = 0.4 = 정확도 가중치 — 엔티티/관계 추출의 정확성.
β = 0.4 = 커버리지 가중치 — 도메인 표현의 폭.
γ = 0.2 = 일관성 가중치 — 논리적 무모순성 (사실 간 충돌 없음).
ALCE Citation Quality
ALCE_Score = |Cite_entailed| / |Cite_total|
Cite_total = 보고서에 달린 전체 인용(citation span) 수.
Cite_entailed = NLI 모델이 인용된 소스가 해당 주장을 entail(함의)한다고 판정한 인용 수.
판정 기준: NLI(citation_span, claim) ∈ {entailment}이면 유효한 인용.

임계값: > 0.85 / 논문: arXiv:2305.14627

UniEval
coherence, consistency, fluency, relevance를 Boolean QA 방식으로 평가. 인간 평가 대비 23% 높은 상관관계.
QuestEval
소스 → 질문 생성 → 요약으로 답변 확인하는 방식. Precision + Recall의 조화 평균.
QuestEval = 2 × (QA_P × QA_R) / (QA_P + QA_R)
QA_R (recall) = 소스에서 생성한 질문 중 요약/보고서가 답할 수 있는 비율 → 커버리지 측정.
QA_P (precision) = 요약에서 생성한 질문 중 정확하게 답변된 비율 → 정확성 측정.
산식: 조화 평균(F1) — Precision과 Recall의 균형. 임계값: > 0.60 권장.
SummaC
NLI 기반 비일관성 탐지. 요약문의 각 문장이 소스와 모순되는지 검사.
BLANC
Reference-free 요약 평가. 요약 조건부 log_prob 향상 측정.
BLANC = log P(D | S) - log P(D | ∅)
D = 원본 문서.
S = 평가 대상 요약/보고서.
log P(D | S) = 요약 S를 컨텍스트로 줬을 때 언어 모델이 D를 생성할 확률의 로그.
log P(D | ∅) = 아무 컨텍스트 없이 D를 생성할 확률의 로그 (베이스라인).
해석: 차이가 클수록 요약이 문서 이해에 유용한 정보를 많이 담고 있음. 참조 답변 불필요.
🌈 다양성 & 정보성 메트릭 ▶
Distinct-n
Distinct-n = |{g : g ∈ G_n}| / |G_n|
G_n = 텍스트에서 추출한 전체 n-gram 리스트 (중복 포함).
|{g : g ∈ G_n}| = 고유(unique) n-gram의 수.
n = n-gram 크기 (보통 n=1 유니그램, n=2 바이그램).
범위: [0, 1]. 1에 가까울수록 어휘 다양성 높음. 단, 긴 텍스트일수록 불리(편향 존재).
Shannon Entropy
H(P) = -Σᵢ p(xᵢ) × log₂(p(xᵢ))
P = 콘텐츠 요소(토픽, 엔티티, 토큰 등)의 확률 분포.
p(xᵢ) = i번째 요소의 출현 확률.
해석: 최대 엔트로피 = log₂(|지지 집합|) (균등 분포). 실제 엔트로피가 이에 가까울수록 다양.
KL Divergence
균등 분포와의 거리. 생성 분포가 얼마나 치우쳐 있는지 측정.
KL(P||Q) = Σᵢ P(xᵢ) × log(P(xᵢ) / Q(xᵢ))
P = 생성 텍스트의 실제 토픽/토큰 분포.
Q = 비교 대상 분포 (보통 균등 분포 U).
해석: KL(P||U)가 클수록 P가 균등에서 벗어남 → 특정 토픽에 치우쳐 있음. 다양성 점수 = 1 - KL/H_max 로 변환하면 [0,1] 범위. > 0.6 권장.
🛡️ 견고성 & 일관성 메트릭 ▶
Cross-Run Consistency
동일 쿼리를 k번 실행했을 때 결과의 의미적 유사도 평균.
CRC = (2 / k(k-1)) × Σᵢ<ⱼ cos_sim(embed(rᵢ), embed(rⱼ))
k = 동일 쿼리에 대한 총 실행 횟수 (권장: 5~10).
rᵢ, rⱼ = i, j번째 실행의 생성 결과.
embed(·) = 문장 임베딩. cos_sim = 코사인 유사도.
목표: > 0.75. 같은 질문에 대해 75% 이상 의미적으로 일관된 답변.
Perturbation Robustness
쿼리를 변형(동의어 교체, 어순 변경 등)했을 때 정확도 유지율.
ECE (Expected Calibration Error)
ECE = Σₘ₌₁ᴹ (|Bₘ| / n) × |acc(Bₘ) - conf(Bₘ)|
M = 신뢰도 구간(bin) 수 (보통 10개).
Bₘ = m번째 신뢰도 구간에 속하는 예측 집합.
|Bₘ|/n = 해당 구간의 비율 (가중치).
acc(Bₘ) = 구간 내 실제 정확도. conf(Bₘ) = 구간 내 평균 신뢰도.
해석: 0에 가까울수록 모델의 자신감과 실제 정확도가 일치 (잘 보정됨).
목표: ECE < 0.1 (잘 보정된 시스템)
Semantic Entropy
다중 생성의 의미 클러스터 엔트로피. 모델이 얼마나 확신하지 못하는지.
SE = -Σᵢ p(cᵢ) × ln(p(cᵢ))
cᵢ = i번째 의미 클러스터 (양방향 NLI entailment로 동치 판정된 응답 그룹).
p(cᵢ) = N회 생성 중 클러스터 cᵢ에 속하는 응답의 비율.
방법: 동일 쿼리로 ~10회 생성 → 의미적 동치 클러스터링 → 엔트로피 계산.
해석: > 0.8 nats = 높은 불확실성 → "근거 부족" 표시 권장.
SE > 0.8 bits → 높은 불확실성 경고
🗣️ 담화 & 일관성 메트릭 ▶
Entity Grid Coherence
문장 간 엔티티 전환 패턴 추적. 논리적 흐름이 자연스러운지 평가.
Centering Theory
담화의 초점(center) 연속성 평가. 주제 전환이 적절한지 측정.
RST (Rhetorical Structure Theory)
수사적 구조 이론 기반. 담화 구조의 계층적 일관성 평가.
⚡ 실시간 탐지 메트릭 ▶
PrefixNLI
생성 도중 실시간으로 비일관성 탐지. 사후 평가가 아닌 생성 중 개입 가능.
장점: 비일관성 발생 즉시 재생성 트리거 가능
SelfCheckGPT
N개 샘플링 → BERTScore 일관성 측정 → 환각 탐지.
hallucination_score(s) = 1 - (1/N) × Σₖ₌₁ᴺ BERTScore(s, sₖ)
s = 평가 대상 문장.
sₖ = 동일 쿼리로 k번째 스토캐스틱 샘플링한 응답에서 s와 가장 유사한 문장.
N = 샘플 수 (권장: 5~20).
BERTScore = 문맥적 임베딩 기반 유사도 (precision/recall/F1).
해석: 많은 샘플에서 나타나는 정보 = 사실적 가능성 높음. 0개 샘플에서만 나타남 = 환각 의심.

메트릭 조합 가이드
여러 메트릭을 하나의 점수로 통합하는 전략

5-1. 조합 전략 비교

전략 산식 장점 단점 적합한 상황
Weighted Sum Σ(wᵢ × mᵢ) 간단, 해석 가능 한 메트릭이 다른 메트릭 보상 가능 일반적 사용
Harmonic Mean n / Σ(1/mᵢ) 최저값에 민감 (비보상적) 0값에 취약 모든 차원 균형 필요 시
Geometric Mean (∏mᵢ)^(1/n) 다른 스케일 처리 0값에 취약 비율 기반 비교
Gate + Weighted ★ if all ≥ θ: Σ(wᵢ×mᵢ) 치명적 실패 방지 경직적 핵심 메트릭 보장 필요 시 (추천)
조합 전략 변수 설명:
wᵢ — 메트릭 i에 대한 가중치. 모든 가중치의 합 Σwᵢ = 1 (정규화 조건)
mᵢ — 개별 메트릭 i의 점수 (정규화 후 [0,1] 범위)
n — 조합에 포함되는 메트릭의 총 개수
∏mᵢ — 모든 mᵢ의 곱 (m₁ × m₂ × … × mₙ)
θ — Gate 임계값. 이 값 미만인 메트릭이 하나라도 있으면 전체 점수를 0으로 처리 (기본 추천: faithfulness θ=0.7, gap_handling θ=0.5)
Weighted Sum: 단순 가중 합산. 한 메트릭의 고점이 다른 메트릭의 저점을 보상(compensate) 가능
Harmonic Mean: 최저값에 강하게 끌림 → 모든 차원이 고르게 높아야 높은 점수. mᵢ=0이면 전체가 0
Geometric Mean: 곱의 n제곱근. 서로 다른 스케일의 메트릭 비교에 적합. mᵢ=0이면 전체가 0
Gate + Weighted ★: 게이트 조건(모든 핵심 메트릭 ≥ θ)을 먼저 통과해야 가중합 계산. 치명적 실패를 원천 차단

5-2. 가중치 결정 방법

⚖️ 5가지 가중치 결정 전략 ▶
  • Kendall's τ / Spearman's ρ: 각 메트릭과 인간 평가의 순위 상관관계 계산 → 상관 높은 메트릭에 높은 가중치
  • AHP (Analytic Hierarchy Process): 쌍대 비교 행렬 → 고유벡터 추출 → 일관성 비율(CR) ≤ 0.1 확인
  • Bayesian Optimization: 가중치 벡터를 하이퍼파라미터로 → 베이지안 최적화로 인간 평가 상관 최대화
  • Learn-to-Rank (LambdaMART): 메트릭=feature, 인간 선호=label → 학습 기반 가중치
  • Bradley-Terry 모델: 쌍대 비교 → 전역 순위 추출 (평가자 신뢰도 자동 반영)

5-3. 정규화 방법

방법산식특성이상치 민감도
Min-Max(X-Xmin)/(Xmax-Xmin)[0,1] 범위 보장높음
Z-Score(X-μ)/σ정규분포 가정강건
Sigmoid1/(1+e^(-k×Z))부드러운 [0,1]낮음 (극단 억제)
Percentile순위 기반비모수적없음
정규화 산식 변수 설명:
X — 정규화할 원래 메트릭 값 (raw score)
Xmin, Xmax — 데이터셋 내 해당 메트릭의 최솟값과 최댓값. Min-Max에서 [0,1] 범위를 결정
μ — 해당 메트릭의 평균값 (population mean)
σ — 해당 메트릭의 표준편차 (population standard deviation). σ=0이면 분모가 0이 되므로 상수 메트릭 제외 필요
Z — Z-Score 정규화 결과값 = (X−μ)/σ. Sigmoid 함수의 입력으로 사용
k — Sigmoid 기울기(steepness) 파라미터. k가 클수록 0/1에 가까운 이진적 분포, k가 작을수록 완만한 S곡선. 추천값: k=1~3
e — 자연상수 (≈ 2.71828)
Min-Max: [0,1] 범위 보장하지만 이상치(outlier)에 의해 대부분 값이 좁은 구간에 몰림
Z-Score: 평균=0, 표준편차=1로 변환. 정규분포 가정 시 최적. 범위가 고정되지 않음
Sigmoid: Z-Score 결과를 [0,1]로 부드럽게 압축. 극단값 억제 효과. 추천 정규화
Percentile: 순위 기반으로 변환 → 분포 형태에 무관. 이상치 영향 없음

5-4. 다중공선성 검사

VIF (Variance Inflation Factor)
VIF = 1 / (1 - R²ⱼ)
VIF — Variance Inflation Factor. 메트릭 j가 다른 메트릭들과 얼마나 공선적(correlated)인지를 나타내는 지표
R²ⱼ — 메트릭 j를 종속변수로 놓고 나머지 메트릭들로 회귀분석했을 때의 결정계수 (0~1). R²ⱼ가 1에 가까우면 메트릭 j는 다른 메트릭들의 선형 조합으로 거의 완벽히 예측 가능 → 중복 정보
해석: VIF = 1 → 공선성 없음 (독립적). VIF = 4 → 메트릭 j의 분산이 75% 설명됨 (조사 필요). VIF = 10 → 90% 설명됨 (심각한 중복 → 제거 또는 PCA 축소)
  • VIF > 4 → 조사 필요
  • VIF > 10 → 심각한 공선성 → PCA로 차원 축소하거나 중복 메트릭 제거

6-1. 권장 복합 평가 산식

def evaluate_research_report(query, sources, report):
    """
    웹서버 지식저장소 deep research 보고서의 품질을 복합 평가하는 함수.

    Args:
        query   (str):  사용자의 원래 검색/질의 텍스트
        sources (list): 검색된 근거 문서 리스트 [{text, url, title}, ...]
        report  (str):  LLM이 생성한 최종 보고서 텍스트

    Returns:
        float: 0.0~1.0 범위의 종합 품질 점수.
               게이트 실패 시 0.0 반환 (치명적 품질 미달).
    """

    # ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    # ❶ 풍부함 (Richness) — Claim Density 방식
    # 보고서가 얼마나 정보 밀도 높은 내용을 담고 있는지 측정
    # ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

    # LLM으로 보고서에서 원자적 주장(atomic claim)을 추출
    # 예: "Python은 1991년에 출시되었다" → 하나의 원자적 주장
    claims = extract_atomic_claims(report)

    # 의미적 유사도 0.85 이상인 주장들을 중복으로 판정하여 제거
    # threshold=0.85: cosine similarity 기준, 높을수록 엄격한 중복 판정
    unique_claims = semantic_deduplicate(claims, threshold=0.85)

    # 100단어당 고유 주장 수 = 정보 밀도 (Claim Density)
    # max(..., 1)로 분모가 0이 되는 것을 방지
    # 해석: richness=3.0이면 100단어당 3개의 고유 정보 포함
    richness = len(unique_claims) / max(word_count(report) / 100, 1)

    # ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    # ❷ 근거 공백 대응 (Gap Handling) — Gap Score
    # 빈 근거일 때 성실하게 대응했는지 측정
    # ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

    # 쿼리에서 다뤄야 할 핵심 측면(aspect)을 추출
    # 예: "Python vs Java 비교" → ["성능", "생태계", "학습곡선", ...]
    required_aspects = extract_required_aspects(query)

    # 보고서에서 실제로 다룬 측면의 수
    covered = count_covered_aspects(report, required_aspects)

    # "정보가 부족하여 확인 불가" 등 명시적으로 갭을 인정한 수
    # 근거 없이 지어내는 것보다 갭을 인정하는 것이 높은 점수
    acknowledged_gaps = count_acknowledged_gaps(report)

    # (다룬 측면 + 인정한 갭) / 전체 필요 측면 → [0, 1]
    # 1.0이면 모든 측면을 다루었거나 못 다룬 이유를 명시
    gap_handling = (covered + acknowledged_gaps) / len(required_aspects)

    # ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    # ❸ 근거 활용 (Grounding) — Faithfulness × RUI
    # 근거를 충실히, 빠짐없이 활용했는지 측정
    # ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

    # RAGAS Faithfulness: 보고서의 각 주장이 소스에 의해 뒷받침되는 비율
    # 높을수록 할루시네이션이 적음. 범위: [0, 1]
    faithfulness = ragas_faithfulness(report, sources)

    # RUI (Retrieval Utilization Index): 제공된 소스 중 실제 활용된 비율
    # utilization = (보고서에서 참조한 소스 수) / (전체 소스 수)
    # 범위: [0, 1]. 낮으면 소스를 무시하고 자체 지식으로 생성한 것
    utilization = retrieval_utilization_index(sources, report)

    # 두 값의 곱 = 충실성 × 활용도. 둘 다 높아야 높은 점수
    # 예: faithfulness=0.9, utilization=0.5 → grounding=0.45 (소스 절반 미활용)
    grounding = faithfulness * utilization

    # ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    # ❹ 반복 탐지 (Repetition) — Semantic Novelty
    # 내용을 반복해서 부풀리는 행위를 감지
    # ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

    # 보고서를 문단 단위로 분리
    paragraphs = split_paragraphs(report)

    # 각 문단이 이전 문단들 대비 얼마나 새로운 정보를 담는지 평균
    # 높을수록 반복이 적음. 범위: [0, 1]
    # 내부: 1 - max(cosine_sim(현재문단, 이전문단들))
    novelty = avg_semantic_novelty(paragraphs)

    # 반복 패널티: novelty가 0.5 미만이면 패널티 부과
    # novelty=0.3 → penalty=0.4 (상당한 반복), novelty=0.8 → penalty=0 (건강)
    repetition_penalty = max(0, 1 - novelty * 2)

    # ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    # 종합 점수 — Gate + Weighted Hybrid 전략
    # ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

    # 게이트 조건: 핵심 메트릭이 임계값 미달이면 즉시 0점 반환
    # faithfulness < 0.7: 할루시네이션 과다 → 보고서 자체가 신뢰 불가
    # gap_handling < 0.5: 필수 측면의 절반도 미다룸 → 보고서 가치 없음
    if faithfulness < 0.7 or gap_handling < 0.5:
        return 0.0  # 게이트 실패 → 재생성 필요

    # 가중 합산: 각 메트릭에 중요도 기반 가중치 적용
    # - richness    (0.30): 정보 밀도가 가장 직접적인 품질 지표
    # - gap_handling(0.20): 근거 공백 대응 (보조 지표)
    # - grounding   (0.30): 근거 활용 충실성 (핵심 신뢰도 지표)
    # - 1 - penalty (0.20): 반복 없음 보너스 (패널티 역수)
    # 가중치 합계: 0.30 + 0.20 + 0.30 + 0.20 = 1.00
    raw_score = (
        0.30 * richness +          # 풍부함 기여분
        0.20 * gap_handling +       # 갭 대응 기여분
        0.30 * grounding +          # 근거 활용 기여분
        0.20 * (1 - repetition_penalty)  # 반복 없음 기여분
    )
    return raw_score  # [0, 1] 범위. 0.7 이상이면 양호한 보고서

⚠️ Goodhart 함정: "평가 메트릭을 게이밍하는 LLM" 핵심 ▶

자율 루프에서 가장 근본적인 위험입니다. LLM이 실제 품질이 아닌 메트릭 점수 자체를 최적화하는 법을 학습합니다.

  • 증상: Faithfulness 점수는 높지만 읽어보면 내용이 빈약. "소스에 따르면…"을 반복하면서 NLI 판정은 통과
  • 근본 원인: 평가 메트릭이 프록시(proxy)이지 실제 목표가 아님
  • 조사 방향:
    • Red-teaming 평가: 의도적으로 메트릭을 속이는 적대적 보고서를 생성해서 평가 파이프라인의 블라인드 스팟 탐색
    • 메타-평가 (Evaluation of Evaluators): 평가 메트릭 자체의 정확도를 주기적으로 인간 판단과 비교 측정. Kendall's τ가 시간에 따라 하락하면 게이밍 징후
    • 논문: Gao et al. "Scaling Laws for Reward Model Overoptimization" (2023) — 보상 모델이 과적합되는 속도를 예측하는 스케일링 법칙
실천 권고: 자율 루프 50회마다 인간 스팟 체크(3~5건 샘플링 + 평가 메트릭과 인간 점수의 순위 상관 비교)를 주기적으로 수행하세요. 상관계수가 떨어지면 메트릭 재캘리브레이션 시점입니다.
🧪 생성 중 실시간 개입 (Runtime Intervention) 핵심 ▶

현재 보고서에서 다룬 메트릭 대부분은 사후 평가(생성 완료 → 평가 → keep/revert)입니다. 하지만 문제가 생기는 시점은 생성 도중입니다.

  • Self-RAG Critique Tokens: [ISREL], [ISSUP], [ISUSE] 토큰을 생성 과정에 삽입하여 실시간 품질 판정. 사후 평가 대비 비용 90%↓ 가능성
  • PrefixNLI: 문장이 완성될 때마다 이전 문장과의 NLI 일관성 검사. 모순 탐지 시 즉시 생성 방향 수정
  • Constrained Decoding: 인용 형식 강제, 최소 근거 개수 강제 등 구조적 제약을 디코딩 단계에서 적용

왜 중요한가: 사후 평가 루프에서 revert가 반복되면 비용이 폭발합니다. 생성 중 개입은 revert 자체를 줄여 루프 효율을 근본적으로 개선합니다.

조사 포인트: Self-RAG의 critique token을 웹서버 지식저장소 맥락에 맞게 커스터마이즈하는 방법. 특히 [ISSUP] (근거 뒷받침 여부) 토큰이 사용자의 ❸ 실패 모드(근거 미활용)에 직접 대응합니다.
📊 pi-autoresearch의 MAD 신뢰도를 평가 파이프라인에 차용 추천 ▶

pi-autoresearch의 Confidence = |improvement| / MAD 접근은 단순하지만 강력합니다. LLM 평가 메트릭에도 동일 원리를 적용할 수 있습니다.

  • 문제: LLM Judge 점수는 동일 입력에도 ±0.1~0.3 변동 (온도, 프롬프트 순서 효과). 개선인지 노이즈인지 구분 불가
  • 적용: 동일 보고서를 3~5회 평가 → MAD 계산 → 개선폭이 MAD의 2배 이상이면 실질적 개선으로 판정
  • 비용: 평가 비용 3~5배 증가하지만, 노이즈에 의한 잘못된 keep/revert를 방지하여 루프 전체 비용 절감 가능

🔬 중기 연구 — 깊이 있는 후속 조사

🏗️ Pareto 기반 다목적 최적화 (Gate+Weighted 너머) ▶

현재 권장하는 Gate+Weighted는 단순하고 효과적이지만, 메트릭 간 트레이드오프를 탐색하지 못합니다.

  • Pareto Front: AlphaEvolve의 MAP-Elites 방식처럼 "풍부함은 높지만 간결성은 낮은" vs "간결하지만 밀도 높은" 다양한 최적 해를 동시에 유지
  • NSGA-II / MOEA: 다목적 진화 알고리즘으로 Faithfulness-Richness-Novelty의 3차원 Pareto Front 탐색
  • 적용 시나리오: 사용자가 "이번엔 상세한 보고서" vs "이번엔 요약형" 같은 선호를 표현할 때, Pareto Front에서 가장 가까운 해를 선택

참고: RoboPhD가 Elo 토너먼트로 검증셋 없이도 진화를 구동한 사례가 있음 — 평가 데이터가 적은 환경에서 유효한 전략

🔄 Bilevel AutoResearch — 메타 최적화 ▶

"평가 파이프라인 자체를 자동으로 개선하는 루프"입니다. 현재는 인간이 메트릭과 가중치를 설계하지만, 이것 역시 자동화 가능합니다.

  • Inner Loop: 보고서 생성 → 평가 → keep/revert (기존 AutoResearch)
  • Outer Loop: 평가 파이프라인의 가중치·임계값·프롬프트를 수정 → 인간 판단과의 상관 측정 → keep/revert
  • 위험: 이중 Goodhart — outer loop도 게이밍될 수 있음. 인간 판단 샘플이 부족하면 과적합
  • 참고: pi-autoresearch 생태계에서 이미 "bilevel autoresearch" 개념이 논의되고 있음
🌡️ Semantic Entropy로 "모르는 것을 아는" 시스템 ▶

섹션 4에서 다룬 Semantic Entropy를 생성 전 필터로 활용하는 방향입니다.

  • 아이디어: 쿼리에 대해 LLM을 5회 샘플링 → 의미 클러스터 수 측정 → 클러스터가 많으면(=높은 불확실성) 해당 부분은 "근거 부족" 라벨링 후 솔직하게 갭 인정
  • 연결: 사용자의 ❷ 실패 모드(빈 근거일 때 내용을 채우지 않음)에 대한 사전 방어. 불확실할 때 지어내는 대신 "확인 불가"라고 명시
  • 구현 비용: 추가 5회 샘플링 비용 발생하지만, 할루시네이션 사후 탐지+revert 비용보다 낮을 수 있음

3 순서 효과 (Position Bias) A/B 비교 시 먼저 제시된 보고서에 높은 점수 평가 순서를 랜덤화하고 양쪽 순서로 2회 평가 후 평균. Pairwise 평가 시 필수
4 길이 바이어스 긴 보고서에 무조건 높은 풍부함 점수 Claim Density(주장/100단어)로 정규화. FActScore의 γ=10 길이 패널티 적용. 절대 길이가 아닌 밀도를 측정
5 벤치마크 오염 평가셋이 LLM 훈련 데이터에 포함되어 있어 평가가 무의미 시간적 분리 (최신 데이터만 평가셋에 사용) 또는 동적 평가셋 생성. SWE-Bench Pro가 이 문제를 SETUPAGENT로 해결한 사례 참고
