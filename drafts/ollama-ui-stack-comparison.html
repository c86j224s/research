8. Homebrew로 설치한 Open WebUI 사용법

brew install --cask open-webui가 canonical Homebrew 설치 명령입니다. 최신 Homebrew에서는 brew install open-webui도 cask로 해석될 수 있지만, 설치되는 것은 CLI 서버가 아니라 Open WebUI Desktop 앱입니다. 그래서 터미널에서 open-webui serve가 바로 되는 형태가 아니며, brew services start open-webui도 일반 사용 경로가 아닙니다. Desktop 앱은 로컬 개인 사용에는 편하지만 초기 Alpha 성격이 있으므로 재현 가능한 서버/상시 운영은 Docker 또는 Python 서버 경로가 더 적합합니다.

로컬 확인: Cask Desktop App

1) Ollama 실행 확인

먼저 모델 서버가 살아 있어야 합니다. 터미널에서 아래를 확인하세요.

ollama list
curl http://127.0.0.1:11434/api/tags

응답이 없으면 ollama serve 또는 brew services start ollama로 시작합니다. Ollama API는 기본적으로 로컬 전용 127.0.0.1:11434로 유지하고, OLLAMA_HOST=0.0.0.0 또는 Docker port publish로 LAN에 열지 마세요.

2) Open WebUI Desktop 열기

터미널 CLI가 아니라 앱을 실행합니다.

open -a "Open WebUI"

또는 Finder/Launchpad에서 Open WebUI 앱을 여세요. 첫 실행 후 첫 계정이 관리자 계정이 됩니다. 네트워크에 노출하기 전에 강한 고유 비밀번호로 관리자 계정을 먼저 만들고, 필요 없으면 추가 가입을 막거나 승인제로 두세요.

3) Ollama 연결 URL

모델이 안 보이면 관리자 설정에서 연결을 확인합니다.

Settings → Admin Settings → Connections → Ollama
http://127.0.0.1:11434

Desktop/로컬 실행은 Docker가 아니므로 보통 host.docker.internal이 필요 없습니다. Desktop 앱이 관리하는 내부 서버의 브라우저 포트는 공식 Homebrew cask 메타데이터에 안정적으로 문서화되어 있지 않으므로, 앱 UI로 여는 것을 기준으로 삼으세요.

브라우저로 접속 가능한 “Docker 서버형”이 목적이면?

Homebrew Desktop 앱은 빠른 로컬 사용에는 편하지만, 상시 서버/다른 기기 접속/운영 배포 목적이면 공식 권장 경로인 Docker 또는 Python 서버 실행이 더 명확합니다. 아래는 로컬 테스트용 quickstart입니다.

docker run -d -p 127.0.0.1:3000:8080 -v open-webui:/app/backend/data --name open-webui ghcr.io/open-webui/open-webui:main

이 방식은 브라우저에서 http://localhost:3000으로 접속합니다. 127.0.0.1:3000:8080 bind는 로컬 테스트용 안전 기본값입니다. 다른 기기에서 접속하게 만들려면 관리자 계정 생성 후 TLS reverse proxy, VPN 또는 IP allowlist, 강한 인증을 먼저 구성하세요. Docker 컨테이너에서 Mac의 Ollama에 붙일 때는 연결 URL이 http://host.docker.internal:11434가 될 수 있습니다. 운영에서는 :main 같은 mutable tag 대신 검토한 release tag 또는 digest pinning을 고려하고, --restart unless-stopped, 고정 WEBUI_SECRET_KEY, 정기 백업을 추가하세요.

Python/uv 서버 설치는 Homebrew cask와 별개

터미널에서 open-webui serve를 쓰고 싶다면 Desktop cask가 아니라 Python 패키지 또는 uv 실행 경로를 사용합니다.

pip install open-webui
open-webui serve

접속 URL은 보통 http://localhost:8080입니다. 임시 실행보다 데이터 위치를 고정하려면 DATA_DIR=~/.open-webui uvx --python 3.11 open-webui@latest serve처럼 DATA_DIR을 지정하세요.

9. “Ollama is running”만 보일 때

브라우저에서 Ollama is running만 보이면 Open WebUI가 열린 것이 아니라 Ollama API 포트 11434를 직접 연 것입니다. 이 화면은 정상적인 Ollama 상태 확인 페이지이지, 채팅 UI가 아닙니다.

진단: Ollama 포트 접속

1) 잘못 연 주소

http://localhost:11434 또는 http://127.0.0.1:11434는 Ollama API입니다.

여기서는 메뉴나 모델 선택 UI가 보이지 않는 것이 정상입니다.

2) Open WebUI 앱 열기

Homebrew cask 설치라면 브라우저 주소를 직접 열지 말고 앱을 실행하세요.

open -a "Open WebUI"

Open WebUI 화면에서는 로그인/회원가입, 새 채팅, 모델 선택 드롭다운이 보여야 합니다. localhost:3000과 localhost:8080도 닫혀 있고 Open WebUI 프로세스가 없다면, Open WebUI가 아직 실행되지 않은 상태입니다.

3) 연결 설정

모델이 안 보이면 관리자 계정으로 들어가서 아래를 확인합니다.

Settings → Admin Settings → Connections → Ollama

설치 방식별 URL을 구분하세요: Homebrew Desktop/Python 서버가 같은 Mac에서 돌면 http://127.0.0.1:11434, Docker Open WebUI가 Mac host의 Ollama에 붙으면 http://host.docker.internal:11434, Docker compose 내부 Ollama라면 compose service/container URL을 씁니다.

저장 후 새 채팅 화면의 모델 드롭다운을 새로고침하세요.

10. Podman으로 브라우저형 Open WebUI 서버 띄우기

Homebrew Desktop 앱이 아니라 브라우저에서 localhost:3000으로 접속하는 서버형 UI를 원하면 Podman으로 Open WebUI 컨테이너만 실행하고, Ollama는 Mac host에서 계속 실행하는 구성이 깔끔합니다.

권장: Open WebUI container + host Ollama

1) Podman 준비

brew install podman
podman machine init
podman machine start

확인: podman version

2) 안전한 로컬 실행

podman run -d --replace --name openwebui -p 127.0.0.1:3000:8080 -v open-webui:/app/backend/data -e OLLAMA_BASE_URL=http://host.containers.internal:11434 ghcr.io/open-webui/open-webui:main

접속: http://localhost:3000

Ollama URL

Podman 컨테이너 안에서 127.0.0.1은 Mac host가 아니라 컨테이너/VM 자신입니다. Open WebUI에는 http://host.containers.internal:11434를 넣으세요.

검증 명령

curl http://127.0.0.1:11434/api/tags
curl http://127.0.0.1:3000
podman exec openwebui curl -fsS http://host.containers.internal:11434/api/tags

기존 open-webui volume에 예전 Ollama URL이 저장되어 있으면 env var보다 UI 저장 설정이 우선될 수 있습니다. 모델이 안 보이면 먼저 Admin Settings → Connections → Ollama에서 URL을 다시 저장하세요. fresh volume은 채팅 기록과 설정을 잃을 수 있으므로, 의도적으로 clean instance를 만들고 백업을 확인한 뒤에만 사용하세요.

피해야 할 것

-p 3000:8080처럼 host IP를 생략하면 LAN에 노출될 수 있습니다. OLLAMA_HOST=0.0.0.0:11434, -p 11434:11434, --network=host는 기본 대안으로 쓰지 마세요.

Podman Compose variant

Compose를 쓰더라도 port host IP를 생략하지 마세요. 아래처럼 loopback bind를 명시합니다.

services:
  openwebui:
    image: ghcr.io/open-webui/open-webui:main
    container_name: openwebui
    ports:
      - "127.0.0.1:3000:8080"
    volumes:
      - open-webui:/app/backend/data
    environment:
      OLLAMA_BASE_URL: http://host.containers.internal:11434
volumes:
  open-webui:

Compose 실행 전 provider가 필요할 수 있습니다: brew install podman-compose. 실행: podman compose up -d 또는 환경에 따라 podman-compose up -d
