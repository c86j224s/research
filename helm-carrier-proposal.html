§ 02 Dynamic Workflows 분석

2026-05-28 Anthropic이 공개한 기능. Claude가 JavaScript 오케스트레이션 스크립트를 동적으로 작성하고, 격리 런타임이 이를 실행하며, 최대 16 concurrent / 1,000 total sub-agent를 병렬 fan-out한다. 중간 결과는 스크립트 변수에 저장되어 Claude의 컨텍스트 윈도우를 오염시키지 않는다.

⚓ 결론

Fleet은 작은~중규모(수~수십 agent)에서 Dynamic Workflows와 사실상 동등하다. 차이는 누가 컨텍스트의 그릇이 되는가다. Workflows는 런타임이, Fleet은 제독이. Helm은 이 그릇을 런타임으로 옮겨, Fleet의 doctrine 안전장치를 유지하면서 1,000-agent 스케일을 가능케 한다.

4.3 핵심 코드 스케치 (개념)

Helm이 동적으로 작성하는 오케스트레이션 스크립트의 형태:

// .fleet/workflows/runs/<id>/script.mjs — Helm이 동적 생성
// Deno 런타임에서 실행: deno run --allow-run=claude --allow-write=.fleet/workflows/ script.mjs

const limit = createSemaphore(16);  // manifest.budget.maxConcurrent 강제

// Phase 1: 다각도 정찰 fan-out
const angles = [
  "src/auth/ 인증 미들웨어 누락 핸들러 탐지",
  "src/routes/ 데코레이터 없는 엔드포인트 탐지",
  "test/ 인증 우회 케이스 커버리지 갭 탐지",
];
const findings = await Promise.all(
  angles.map(a => limit(() => dispatchWorker(a, { isolation: "cwd+env" })))
);

// Phase 2: adversarial cross-check (validation phase = quorum 2/3)
const verified = await Promise.all(
  findings.map(f => limit(() =>
    dispatchWorker(`이 발견을 비판적으로 반증 시도: ${JSON.stringify(f)}`)
  ))
);

// 최종 합성만 carrier_jobs로 반환
return aggregate({
  confirmed: verified.filter(v => v.confirmed),
  rejected: verified.filter(v => !v.confirmed),
});

dispatchWorker()는 Helm이 Deno global scope에 주입하는 API로, 내부적으로 claude -p를 격리 환경에서 spawn한다.

4.4 WorkflowManifest 스키마

interface WorkflowManifest {
  budget: {
    maxAgents: number;        // 누적 상한 (≤ 1000)
    tokensPerAgent: number;   // 1개 agent 토큰 예산
    maxConcurrent: number;    // 동시 상한 (≤ 16, 런타임 강제)
  };
  toolAllowlist: string[];    // EXECUTOR_MCP_TOOL_IDS 하위 집합
  domainAllowlist: string[];  // 외부 접근 허용 도메인
  mutationPhases: PhaseId[];  // 파일 쓰기/git 포함 phase
  phases: PhaseDefinition[];
}

interface PhaseDefinition {
  id: PhaseId;
  type: 'analysis' | 'mutation' | 'validation';
  failurePolicy: 'fail-fast' | 'best-effort' | { quorum: number };
}

D2 — 샌드박스 모델

✓ 권장안

Deno — --allow-run=claude --allow-write=.fleet/workflows/

탈락 옵션

D4 — 컨텍스트 격리 보장 (3-Layer)

✓ 권장안 — 3-Layer 격리

  2. Env 화이트리스트: process.env 통과 금지, 명시 항목만

CarrierSessionStore JSONL이 infra.dataDir에 공유되므로, pool 진입 시 부모 sessionId 교차오염 가능.
